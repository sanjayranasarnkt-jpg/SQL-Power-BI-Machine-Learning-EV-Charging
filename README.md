# ⚡ EV Charging Network Analysis (Capstone Project)

## 📊 Project Overview

This project analyzes the **Electric Vehicle (EV) Charging Network in Warsaw, Poland** using **SQL, Power BI, and Machine Learning**.

The goal is to uncover:

* Charging demand patterns
* Revenue opportunities
* Infrastructure gaps
* Customer behavior insights

---

## 🚀 Key Metrics

* 🔌 Total Sessions: **114,000+**
* ⚡ Total Energy: **4M+ kWh**
* 🏢 Stations: **65**
* 🔌 Charging Plugs: **390**
* 👥 Customers: **8,000+**
* 💰 Total Revenue: **zł11.4M**

---

## 🧠 Project Components

### 1️⃣ SQL Analysis

* 14 Queries (Basic → Advanced)
* Data exploration, joins, aggregations
* Revenue & energy analysis
* Infrastructure gap analysis

#### 🔍 Key Findings

* All **18 districts are under-equipped** 
* 5 districts have **zero charging stations** 
* ~95 EVs per plug → **huge infrastructure gap** 
* High-income users pay **~89% more per session** 

---

### 2️⃣ Power BI Dashboard

4-page interactive dashboard:

* 📍 Infrastructure Overview
* ⚡ Charging Usage & Demand
* 👥 Customer Behavior
* 💰 Revenue & Planning

#### 📈 Insights

* Mid-range areas dominate usage
* Tuesday = busiest day
* 8 AM–3 PM = peak usage window 
* Ursynów & Wesoła generate **36% revenue** 

---

### 3️⃣ Machine Learning Models

#### 🔮 Demand Forecasting

* Model: Random Forest
* R² Score: **0.87**
* Predicts future charging demand

#### 💸 Cost Prediction

* Model: Gradient Boosting
* R² Score: **0.91**
* Key driver: kWh consumed

#### 👥 Customer Segmentation

* Model: K-Means Clustering
* 3 segments:

  * 💎 High-value users
  * 📅 Regular commuters
  * 🔋 Occasional users

---

## 📊 Key Business Insights

* 🚨 Massive infrastructure gap (34K plug shortage) 
* 💰 Revenue directly linked to station availability
* 📈 High-income areas = untapped revenue potential
* ⚡ Demand is consistent → daily usage behavior

---

## 💡 Strategic Recommendations

* 🔌 Build stations in zero-coverage districts
* 💰 Expand infrastructure in high-income areas
* 📊 Use ML forecasting for planning
* 🎯 Launch subscription plans for commuters
* 🕒 Implement dynamic pricing (peak hours)
